📋 YAML Configuration Reference

Only the keys listed below are processed by this app. Any unsupported keys will be IGNORED.

═══ VALID OPERATIONS (set enabled: true/false) ═══

  • ✅ smart_type_conversion — Auto detect & convert data types [methods: auto, conservative, aggressive]
  • ✅ handle_missing — Handle missing/null values [methods: auto, remove, fill_median, fill_mode]
  • ✅ remove_duplicates — Remove duplicate rows
  • ✅ feature_engineering — Create derived features (date/text/numeric)
  • ✅ clean_text — Clean text formatting (whitespace, title case)
  • ✅ handle_outliers — Handle outliers [methods: cap, remove, ignore]
  • ✅ remove_high_missing — Remove columns >50% missing ⚠️
  • ✅ redact_phi — Redact Protected Health Information ⚠️
  • ✅ validate_clinical — Validate clinical measurement ranges
  • ✅ standardize_codes — Standardize medical codes (ICD/CPT)
  • ✅ validate_ranges — Validate numeric value ranges (3×IQR clip)

═══ SAMPLE YAML ═══

operations:
  smart_type_conversion:
    description: "Convert data types"
    enabled: true
    methods: [auto, conservative, aggressive]
    default_method: "auto"
  handle_missing:
    description: "Handle missing values"
    enabled: true
    methods: [auto, remove, fill_median, fill_mode]
    default_method: "auto"
  remove_duplicates:
    description: "Remove duplicates"
    enabled: true
  clean_text:
    description: "Clean text formatting"
    enabled: true
  handle_outliers:
    description: "Handle outliers"
    enabled: true
    methods: [cap, remove, ignore]
    default_method: "cap"
  validate_ranges:
    description: "Validate numeric ranges"
    enabled: true
