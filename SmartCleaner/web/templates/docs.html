Getting Started

Medicl is an advanced healthcare-centric data preprocessing platform designed to handle structured, semi-structured, and unstructured medical records with intelligent parsing and HIPAA-compliant redaction.

Installation

Run Medicl locally using Docker or standard Python environments.

Docker (Recommended)

The easiest way to get started is using Docker Compose.

docker-compose up -d
# Open http://localhost:5000
Local Python Setup

Ensure you have Python 3.10+ installed.

pip install -r requirements.txt
python server.py

Interface Overview

The Web UI is divided into three functional phases to guide you through the data cleaning process.

🔍 Data Analysis

Upload your dataset and get an immediate AI-powered quality report, missing value heatmaps, and schema mapping suggestions.

⚙️ Review & Adjust

Fine-tune exactly which cleaning operations to perform. Toggle HIPAA redaction, outlier clipping, and medical code standardization.

🚀 Execute & Export

Run the pipeline and view side-by-side "Before/After" metrics. Export your cleaned data in CSV, XLSX, JSON, or Parquet.

Cleaning Operations

Medicl provides 11 specialized operations tailored for medical and technical data cleaning.

Smart Type Conversion

Auto-detects intended data types and fixes type mismatches in columns.

PHI Redaction

Redacts names, phones, IDs, and emails to ensure HIPAA/GDPR compliance.

ICD-10 Mapping

Aligns clinical diagnoses with standard ICD-10 registry codes using fuzzy matching.

Outlier Handling

Detects and handles anomalous values using Z-score or IQR methods.

PHI Redaction Notice

While Medicl's PHI redaction is robust, it is provided on a best-effort basis. Always independently verify output before sharing sensitive clinical data.

API Reference

Our RESTful API allows you to integrate Medicl's cleaning pipeline into your own clinical workflows and applications.

Authentication & Sessions

The API uses session-based tracking. A session is automatically created upon your first valid file upload.

Upload Endpoint

POST /api/upload
Content-Type: multipart/form-data

Python Example

import requests

files = {'file': open('patient_records.csv', 'rb')}
r = requests.post('http://localhost:5000/api/upload', files=files)
session_id = r.json()['session_id']

Execute Endpoint

POST /api/execute
Content-Type: application/json

Example Payload

{
  "session_id": "uuid-here",
  "operations": {
    "redact_phi": { "checked": true },
    "remove_duplicates": { "checked": true },
    "smart_type_conversion": { "checked": true }
  }
}

Export Endpoint

Retrieve the cleaned data once processing is complete.

GET /api/export/{session_id}?format=csv

Frequently Asked Questions

Question Answer
Is my data stored on a server? No. Medicl processes data 100% locally in-memory. Once the session ends, data is purged.
What is the maximum file size? The default limit is 50MB, but this can be adjusted in the server.py config.
Do you support PDF clinical notes? Yes, via the "Unstructured" tab, which uses Docling and OCR to parse clinical text.
